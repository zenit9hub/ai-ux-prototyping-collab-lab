# AI Agent UI/UX Prototyping · 공동 실습 저장소

이 저장소는 AI 에이전트를 활용한 UI/UX 디자인 프로토타이핑을 **실무와 유사한 협업 흐름**으로 연습하고, 교육 후 현업에 재사용할 수 있도록 만든 GitHub 기반 작업 공간입니다.

## 시작점

- 운영 원칙과 역할: [`00. 운영·공통 기준/00. 통합 R&R 및 전체 작업 흐름.md`](<00. 운영·공통 기준/00. 통합 R&R 및 전체 작업 흐름.md>)
- AI 에이전트 작업 규칙: [`AGENTS.md`](AGENTS.md)
- 단계별 품질 게이트: [`00. 운영·공통 기준/02. 산출물 계약·품질 게이트.md`](<00. 운영·공통 기준/02. 산출물 계약·품질 게이트.md>)
- 시각 흐름 안내: [`00. 운영·공통 기준/visual/00. AI Native Workflow Map.html`](<00. 운영·공통 기준/visual/00. AI Native Workflow Map.html>)

## 작업 흐름

```text
01. 요구사항 수집·리서치
  → 02. PRD
  → 03. IA
  → 04. Wireframe
  → 05. Hi-Fi Design (Figma Make, Claude Design)
  → 06. 개발 전달·퍼블리싱
  → 07. 배포·검증 (GitHub Pages)
```

각 단계는 직전 단계에서 **승인된 baseline**만 입력으로 사용합니다. 초안은 다음 단계의 사실이 될 수 없으며, `accepted` 상태가 된 산출물만 다음 단계로 전달합니다.

## 이 저장소의 역할

강의 Vault는 이론·예시·강의 진행을 위한 **학습 기준 자료**입니다. 이 저장소는 그 지식을 실제 업무처럼 수행하는 **작업 시스템**입니다. 따라서 강의 자료의 개인 경로나 대형 원본 파일을 복사하지 않고, 필요한 기준은 여기의 공통 운영 문서와 단계별 계약으로 독립적으로 유지합니다.

## 빠른 시작

1. `00. 운영·공통 기준`의 R&R, 하네스, 품질 게이트를 읽습니다.
2. 담당 사례의 `01. 요구사항 수집·리서치`에서 증거와 가설을 분리합니다.
3. 단계별 `README.md`의 입력·출력·완료 기준을 충족한 뒤 Pull Request로 검토를 요청합니다.
4. 다음 단계 담당자는 병합된 `main`의 accepted baseline만 사용합니다.

## 운영 원칙

- 폴더는 개인 권한이 아니라 산출물의 성숙 단계를 나타냅니다.
- 강사는 공통 기준·코칭·품질 게이트를 안내하지만, 사업 사실이나 현업 의사결정을 대신 승인하지 않습니다.
- AI 에이전트는 제약된 초안과 변형안을 생성할 수 있으나 승인 주체가 아닙니다.
- Hi-Fi의 기준 원본은 `05`에 고정하고, `06`은 복사본이 아닌 전달 manifest로 연결합니다.
- 실제 프론트엔드 소스 코드는 기본적으로 별도 개발 저장소에서 관리합니다.
- 07. 배포·검증 (GitHub Pages)은 교육·협업 검증용 정적 웹 결과를 자동 배포합니다. 실제 제품 코드의 운영 배포를 대체하지 않습니다.

## 상태 표기

`draft` → `review` → `accepted` → `superseded`

각 canonical 산출물에는 최소한 `case_id`, `stage`, `owner`, `reviewer`, `upstream_baseline`, `next_consumer`를 기록합니다. 자세한 형식은 템플릿을 따릅니다.
