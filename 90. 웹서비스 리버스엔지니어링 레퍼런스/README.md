# 90. 웹서비스 리버스엔지니어링 레퍼런스

이 영역은 공개 웹서비스의 화면과 흐름을 관찰해 UX 구조·업무 가설·검증 질문을 추상화하는 학습용 레퍼런스다. 특정 서비스를 복제하거나 그 서비스의 실제 정책·기술 구현을 단정하는 데 목적이 있지 않으며, 관찰 결과를 수강생 자신의 프로젝트에 맞는 요구사항·IA·Wireframe·Q&A로 다시 해석하는 데 사용한다.

> [!warning] 01~07의 다음 단계가 아니다
> `90`은 선형 실무 파이프라인과 분리된 `reference-only` 학습 영역이다. 여기서 `accepted`가 된 문서도 특정 서비스의 실제 정책이나 `01. 요구사항 수집·리서치`의 accepted input으로 자동 승격되지 않는다.

## 수강생 사용 흐름

1. **서비스 선정** — 공개 접근이 가능한 서비스 URL과 관찰 목적을 정한다.
2. **범위·권리 확인** — 약관, `robots.txt`, 로그인·결제·개인정보 영역 여부를 확인하고 수집 방식을 결정한다.
3. **관찰 근거 기록** — 메뉴, 화면, 과업, 상태, 예외를 URL·관찰 시각·근거 ID와 함께 기록한다.
4. **역설계 문서화** — 관찰한 사실을 PRD·IA·Wireframe 관점으로 재구성하되, 해석과 미확정을 분리한다.
5. **시니어 Q&A** — 운영 규칙, 권한, 실패, 데이터 신뢰성처럼 화면만 봐서는 알 수 없는 항목을 질문으로 남긴다.
6. **현업 적용** — 원래 서비스의 기능·브랜드·카피를 베끼지 않고, 자신의 문제·사용자·제약에 맞는 적용 포인트만 `01~07`의 새 사례로 옮긴다.

## 시작 전 반드시 확인할 것

- 대상은 로그인 우회 없이 접근 가능한 공개 페이지인가?
- 관찰 목적은 UX·업무 패턴의 학습인가, 서비스·자산·코드의 복제인가?
- 자동 수집이 필요한가, 아니면 브라우저 관찰이 더 안전하고 충분한가?
- 출처 URL, 관찰일, 화면 상태, 수집 방식을 각 근거에 기록할 수 있는가?
- 개인정보, 고객 정보, 결제, 계정 설정, 비공개 API, 대량 데이터가 관찰 범위에 포함되지 않는가?

## 이 영역에서 하지 않는 일

- 로그인·결제·권한·차단을 우회해 수집하지 않는다.
- 소스코드, API endpoint, 비공개 데이터, 브랜드 자산, 원문 카피를 복제·재배포하지 않는다.
- 관찰한 화면만으로 실제 사업 정책, 데이터 처리, 권한 규칙, 기술 구조를 사실처럼 선언하지 않는다.
- 외부 서비스 사례를 자신의 `01~07` 단계 산출물에 근거 없이 직접 붙여 넣지 않는다.
